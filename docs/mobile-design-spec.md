# 移动端设计方案 v2

## 设计原则
- **基于使用场景**：手机竖屏（日常使用）、手机横屏（导航模式）、平板（类 PC 体验）
- **不是缩小 PC**：手机端是独立的信息架构，不是三栏的等比缩放
- **地图为背景**：行程信息和对话浮于地图之上，地图始终可见
- **信息密度适中**：仪表盘式紧凑布局，一屏看到关键信息

## 参考应用分析
| 应用 | 设计模式 | 可借鉴 |
|------|---------|--------|
| 高德地图 | 底部 tab（首页/出行/发现/我的），地图全屏 | 地图优先 + 底部导航 |
| 携程 | 底部 tab（首页/行程/旅拍/消息/我的） | 行程按天规划 |
| Google Maps | 地图全屏 + 底部 sheet 展开详情 | 浮层交互模式 |
| 马蜂窝 | 内容优先，行程笔记 | 丰富行程展示 |

## 手机竖屏布局（< 768px）

### 顶部 Header（44px）
```
┌─────────────────────────────────┐
│ [Logo] 随心自驾     [消息] [设置] │
└─────────────────────────────────┘
```
- Logo 28x28 + 应用名
- 右侧：消息通知（带角标）+ 设置入口

### 主内容区（地图全屏背景）
地图始终渲染在底层，行程和对话信息以**浮层/sheet**形式覆盖在地图上方。

### 底部 Tab 栏（56px + safe area）
```
┌─────────────────────────────────┐
│  [行程]    [地图]    [对话]     │
└─────────────────────────────────┘
```
- 3 个 tab：行程（主功能）、地图（全屏查看）、对话（AI 助手）
- 图标 + 文字，赤陶色激活态

### 行程 Tab（浮层覆盖地图，可上下滑动）

## 交互设计

### 地图交互
| 操作 | PC 端 | 手机端 |
|------|------|--------|
| 查看 POI | 双击 | 点击底部信息条展开 |
| 添加途经点 | 双击弹浮层 | 地图 tab 下长按 |
| 切换策略 | 策略面板 | 行程 tab 内切换 |
| 归位 | 归位按钮 | 归位按钮 |
| 卫星切换 | 卫星按钮 | 卫星按钮 |

### 行程规划流程
1. 用户输入起终点 → 点击"进入规划"
2. 默认显示"总览"tab（路线概览 + 策略）
3. 用户可切换 Day1-N 查看每天详情
4. 策略切换实时更新路线
5. 搜索景点 → 选点 → 加入行程

## 响应式断点
| 断点 | 宽度 | 布局 |
|------|------|------|
| 手机竖屏 | < 768px | 底部 tab + 浮层 |
| 平板竖屏 | 768-1023px | 侧栏可收起 |
| 平板横屏/小笔记本 | 1024-1279px | 三栏压缩 |
| 笔记本 | 1280-1439px | 三栏标准 |
| 桌面 | 1440px+ | 三栏宽松 |
| 大屏 | 1920px+ | 三栏加宽 |

## 配色（沿用 Organic）

- 卡片：`#FFFFFF`（浮层/sheet）
- 主色：`#C66B3D`（赤陶，按钮/激活态）
- 辅助：`#606C38`（苔绿，起点/成功态）
- 文字：`#2D2A26`（主）、`#8B8578`（次）
- 跳色：`#7EB8DA`（雾蓝）、`#E8C547`（琥珀）、`#E8A598`（柔珊瑚）
