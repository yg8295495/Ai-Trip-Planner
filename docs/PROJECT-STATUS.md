## 2. 待解决问题

### P0：核心功能

#### 2.1 走廊发现（Phase 3）
**设计方案：高德为主 + AI 辅助**

```
Step 1: 获取最优路径
  高德 Driving API → 路线 polyline + 总时长
  → 地图显示直达路线

Step 2: 询问偏离距离
  AI 问用户："你能接受偏离最优路径多远？（如 20km、50km）"
  用户回答后，前端记录 maxDeviation

Step 3: 搜索沿途景点
  沿路线 polyline 采样多个点（每隔 N km 取一个点）
  → 每个点调用高德周边搜索 API（半径=maxDeviation）
  → 合并去重 → 得到沿途景点列表
  → 获取每个景点的详情和图片（高德 POI 详情 API）

Step 4: 展示景点
  地图上显示所有候选景点（蓝色虚线圈表示偏离范围）
  每个标记有信息卡片（名称 + 图片 + 简介 + 评分）
  用户点击选择感兴趣的兴趣点（高亮为绿色）

Step 5: 确认并重算
  用户点"确认" → 选中景点作为途经点
  → 重新调用高德 Driving API（带途经点）
  → 更新地图路线 + 行程
```

**实现要点：**
- 路线获取、景点搜索、图片获取 → 高德 REST API
- 推荐逻辑、与用户交互、格式化输出 → AI
- AI 不需要"知道"哪些景点好，高德提供数据，AI 负责问用户问题和整理结果

**高德 API 依赖：**
- `v3/direction/driving` — 驾车路线规划（已有）
- `v3/place/around` — 周边搜索（待接入）
- `v3/place/detail` — POI 详情 + 图片（待接入）

## 8. 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
