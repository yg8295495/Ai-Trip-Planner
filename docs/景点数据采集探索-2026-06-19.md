# 景点数据采集探索（2026-06-19~20）

## 数据源优先级
1. **携程**（主力）：城市聚合页REST API，每城市几千景点结构化数据
2. **高德**（标准底册）：5A/4A/世界遗产1941条，名称校准+坐标锚定
3. **马蜂窝**（补充）：少数超级景点的子景点/tips/评论，WAF限制大
4. **OSM**（兜底）：冷门野生景点

## 携程 REST API
- 端点：`POST m.ctrip.com/restapi/soa2/18109/json/getAttractionList`
- 参数：districtId(城市ID), index(页码), count(20), scene:"online", sortType:1
- 返回：totalCount, hasMore, attractionList[{poiName, commentScore, commentCount, zoneName}]
- 城市ID映射：从携程首页链接提取2258个（/place/pinyin{ID}.html → ID=districtId）
- 效率：curl直接调3秒/页，31城市约1-2小时

## 采集字段
- 基础：poiId, poiName, commentScore, commentCount, zoneName
- 详情：ticketPrice, openTime, distanceFromCenter
- 标签：5A/4A, 分类（亲子/观光/名胜古迹）, 热门榜（必打卡/100强）

## 噪音过滤（前期排除）
- 名称含"演出""话剧""相声""表演""门票""索道""观光车" → 排除
- 分类为纯演出/票务类 → 排除
- 只保留：景点/景区/公园/博物馆/古镇/遗址/山/湖/岛

## 图片策略
- 携程/高德/马蜂窝：存URL不存文件
- 顶级景点：摄影合集图册（网络URL展示）

## 马蜂窝页面形态（6种）
1. Banner精准匹配（颐和园、泰山）
2. 卡片多结果（故宫"地点"区域）
3. 小图集+sharejump（格萨尔王城）
4. 目的地型（黄果树，type=10）
5. 相关地点卡片（云龙涧"相关地点"区域）
6. 无匹配（坪山樱花谷）

## 关键技术发现
- Chrome 149加`--user-data-dir`才启用remote debugging
- CDP连用户Chrome绕过WAF，需5-10秒间隔+心跳保活
- 携程__NEXT_DATA__含817KB数据，attractionList数组有景点列表
- 携程翻页API：getAttractionList + index参数
- 脚本做不了语义匹配，需AI判断
- LLM-index项目RSC payload提取方法论可复用
