# 移动端交互设计方案 v4

> 基于用户反馈重设计：**地图是主角**，起终点默认可见，AI 通过通知引导交互。

---

## 1. 核心设计原则

### 原则
- **地图是主角**：始终全屏，是主要交互界面
- **起终点默认可见**：打开 app 就能看到输入框，不需要点按钮
- **AI 是通知式**：不是固定页面，而是通过通知引导用户查看
- **行程是覆盖层**：从底部滑出，不打断地图交互

### 与 PC 端的区别
| 维度 | PC 端 | 移动端 |
|------|------|--------|
| 布局 | 三栏并排 | 地图全屏 + 底部面板 |
| 起终点 | 侧栏表单 | 底部面板内嵌 |
| AI | 左侧固定面板 | 右上通知 + 浮窗 |
| POI | 列表 + 地图标记 | 地图标记为主 |
| 行程 | 右侧固定栏 | 底部可滑动面板 |

## 2. 用户完整流程

### 流程 1：规划新行程
```
1. 打开 app
   ↓
2. 看到地图 + 底部面板（起点/终点输入框默认可见）
   ↓
3. 输入起终点 → 路线实时显示在地图上
   ↓
4. 点击"规划行程" → 底部面板切换到路线选择
   ↓
5. 显示 3 条路线（策略 2/13/10），参考高德/百度
   ↓
6. 用户切换对比路线（不触发 API，纯前端切换）
   ↓
7. 点击"确定" → 弹出偏好填写面板：
   ┌──────────────────────────┐
   │ 旅行天数: [7] 天          │
   │ 每日驾驶上限: [5] h       │
   │ [确认]                    │
   └──────────────────────────┘
   ↓
8. 确认偏好 → 触发 API 调用：
   - 高德 polygon 搜索沿途景点
   - 同时组装系统提示词 + 路线信息 + 途径城市
   - 发送给大模型
   ↓
9. 地图上显示 POI 标记
   ↓
10. 用户查看景点（不需要 AI 也能完成）
```

### 流程 2：AI 推荐景点（可选，与流程 1 并行）
```
1. 用户确认路线 + 偏好后
   ↓
2. 系统同时：
   - 调用高德 API 搜索沿途景点
   - 组装系统提示词 + 路线数据 + 途径城市
   - 发送给大模型
   ↓
3. 大模型分析后生成推荐景点列表（JSON 格式）
   ↓
4. 右上角弹出半透明通知：
   ┌──────────────────┐
   │ 🤖 AI 为你推荐了  │
   │ 3 个沿途景点      │
   │ [点击查看]        │
   └──────────────────┘
   ↓
5. 用户正在查看地图上的景点标记
   ↓
6. 随时可以点击通知 → 进入对话界面
   ↓
7. AI 展示推荐（先说理由，再给选择）：
   ┌──────────────────────────────────┐
   │ 🤖 我为你推荐了 3 个景点：       │
   │                                  │
   │ 📍 推荐理由：                    │
   │ • 当前季节适合户外                │
   │ • 天门山距路线最近（2.3km）       │
   │ • 黄龙洞是溶洞奇观，四季可游     │
   │                                  │
   │ ☑ 🏔️ 天门山景区                 │
   │   距路线 2.3km | ⭐4.8 | 适合登山│
   │                                  │
   │ ☑ 🕳️ 黄龙洞                     │
   │   距路线 5.1km | ⭐4.6 | 平缓    │
   │                                  │
   │ ☐ 🌊 宝峰湖                     │
   │   距路线 8.2km | ⭐4.5 | 有老人  │
   │                                  │
   │ ──────────────────────────────── │
   │ 💬 我先不选，想先聊聊            │  ← 其他选项
   │                                  │
   │ ──────────────────────────────── │
   │ 📝 补充偏好：                    │
   │ ☑ 平缓路线                      │  ← 景点类型偏好
   │ ☐ 爬山登山                      │
   │ ☑ 适合老人小孩                  │
   │                                  │
   │ [确认选择]                        │
   └──────────────────────────────────┘
   ↓
8. 用户操作：
   - 选项 A：多选景点 → 确认 → 提交答复
   - 选项 B："我先不选" → 继续对话，AI 根据偏好重新推荐
   ↓
9. 自动返回主页面 → 地图上显示新标记
   ↓
10. AI 头像动效（等待回复）
   ↓
11. AI 生成按天规划 → 新通知引导查看
```

### 流程 3：不用 AI 的完整流程
```
1. 输入起终点 → 选择路线 → 填写偏好
   ↓

   ↓
3. 点击标记 → 查看详情 → 加入行程
   ↓
4. 按天规划 → 编辑调整
   ↓

（全程不需要 AI 参与）
```
   │              [确认选择]   │
   └──────────────────────────┘
   ↓
6. 用户多选确认 → 相当于提交答复
   ↓
7. 自动返回主页面 → 地图上显示新标记
   ↓
8. AI 头像动效（等待回复）
   ↓
9. AI 生成按天规划 → 新通知引导查看
```

### 流程 3：编辑行程
```
1. 点击底部面板展开
   ↓
2. 看到按天规划（Day1 | Day2 | Day3 ...）
   ↓
3. 点击某天 → 看到当天景点列表
   ↓
4. 长按景点 → 拖拽调整顺序
   ↓
5. 滑动删除景点
   ↓
6. 点击景点 → 地图聚焦 + 详情
```

## 4. 交互设计

### 4.1 底部面板状态机
```
[初始] → 输入起终点 → [待规划]
[待规划] → 点击"规划行程" → [路线选择]
[路线选择] → 点击"确定" → [偏好填写]
[偏好填写] → 确认偏好 → [规划中]（触发 API + AI）
[规划中] → 查看景点 → [景点选择]
[景点选择] → 确认选择 → [规划完成]
[规划完成] → 点击某天 → [日详情]
```

### 4.2 AI 交互状态机
```
[空闲] → AI 有新推荐 → [通知中]
[通知中] → 用户点击 → [对话中]
[对话中] → 用户发送 → [等待回复]
[等待回复] → AI 回复 → [回复中]
[回复中] → 景点推荐 → [景点选择]
[景点选择] → 用户操作：
  ├→ 多选确认 → [已确认] → 返回主页面
  └→ "先不选" → [对话中] → AI 根据偏好重新推荐
```

### 4.3 手势操作
| 手势 | 效果 |
|------|------|
| 单击地图 | 无操作（避免误触） |
| 长按地图 | 添加途经点 |
| 双击地图 | 放大 |
| 捏合 | 缩放 |
| 上滑底部面板 | 展开更多内容 |
| 下滑底部面板 | 收起 |
| 点击 POI 标记 | 弹出详情 |
| 左滑 POI 标记 | 加入行程 |
| 右滑底部面板内列表项 | 删除 |

## 5. 组件架构

### 5.1 新增组件
| 组件 | 职责 |
|------|------|
| `MobileMapLayout.vue` | 移动端主布局：地图 + 底部面板 + AI 通知 |
| `BottomSheet.vue` | 可滑动底部面板容器 |
| `RouteSelector.vue` | 路线选择面板（3 条路线，切换不触发 API） |
| `PreferenceForm.vue` | 偏好填写弹窗（天数/驾驶时长，确认后触发 API） |
| `PoiMarker.vue` | 地图上的 POI 标记 |
| `PoiDetailCard.vue` | POI 详情弹窗 |
| `AiNotification.vue` | AI 通知弹窗（右上角） |
| `AiRecommendDialog.vue` | AI 景点多选确认对话框 |
| `AiAvatar.vue` | AI 头像 + 等待动效 |

### 5.2 修改组件
| 组件 | 改动 |
|------|------|
| `MapPanel.vue` | 移动端：全屏 + 路线选择高亮 |
| `ItineraryStrip.vue` | 移动端：改为底部面板内容 |
| `ChatPanel.vue` | 移动端：对话覆盖层 + 景点选择 |

### 5.3 数据流
```
用户输入起终点 → tripStore.params（实时显示路线）
选择路线 → tripStore.currentStrategy（切换 polyline，不触发 API）
点击确定 → 弹出 PreferenceForm（天数/驾驶时长）
确认偏好 → 触发并行处理：
  ├→ 高德 polygon 搜索沿途景点 → tripStore.candidatePois → 地图标记
  └→ 组装系统提示词 + 路线 + 途径城市 + 偏好 → 发送大模型
大模型回复 → tripStore.aiRecommendations → AI 通知（右上角）
用户查看 AI → 对话界面 → 展示推荐理由 + 景点选择 + 偏好收集
用户操作：
  ├→ 多选景点确认 → tripStore.selectedPois → 返回主页面
  └→ "先不选" → 继续对话 → AI 根据偏好重新推荐
```
